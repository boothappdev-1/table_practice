HTML Tables

The way to make a table of data in HTML is to use the <table></table> element. And, similar to how we used an li for each list item within a ul (unordered list), we use a tr (table row) for each row within a table:

    <table>
      <tr></tr>
      <tr></tr>
    </table>

But that would just produce empty rows. For each cell in the row, you need to add a <td> (table data) element:

    <table>
      <tr><td>Andrew Luck</td><td>Quarterback</td></tr>
      <tr><td>Wes Welker</td><td>Wide Receiver</td></tr>
      <tr><td>Matt Forte</td><td>Running Back</td></tr>
    </table>

which produces

Andrew LuckQuarterback
Wes WelkerWide Receiver
Matt ForteRunning Back

You need to have the same number of tds, or cells, in each row or your table is going to look wonky.

Usually in our code we prefer to keep each element on its own line, for our own sanity, unless it is a simple inline element like <em> or <small> or <span>; so tables usually look like this:

    <table>
      <tr>
        <td>Andrew Luck</td>
        <td>Quarterback</td>
      </tr>
      <tr>
        <td>Wes Welker</td>
        <td>Wide Receiver</td>
      </tr>
      <tr>
        <td>Matt Forte</td>
        <td>Running Back</td>
      </tr>
    </table>

You can also use <th> (table heading) elements instead of <td> for cells which represent column headings:

    <table>
      <tr>
        <th>Player</th>
        <th>Position</th>
      </tr>
      <tr>
        <td>Andrew Luck</td>
        <td>Quarterback</td>
      </tr>
      <tr>
        <td>Wes Welker</td>
        <td>Wide Receiver</td>
      </tr>
      <tr>
        <td>Matt Forte</td>
        <td>Running Back</td>
      </tr>
    </table>

which produces

Player Position
Andrew Luck Quarterback
Wes Welker Wide Receiver
Matt Forte Running Back

Finally, and optionally, you can use <thead> to group rows containing headings and <tbody> to group rows containing data:

    <table>
      <thead>
        <tr>
          <th>Player</th>
          <th>Position</th>
        </tr>
      </thead>
      <tbody>
        <tr>
          <td>Andrew Luck</td>
          <td>Quarterback</td>
        </tr>
        <tr>
          <td>Wes Welker</td>
          <td>Wide Receiver</td>
        </tr>
        <tr>
          <td>Matt Forte</td>
          <td>Running Back</td>
        </tr>
      </tbody>
    </table>

The last thing to note is, if you need to "merge" two cells together, you can make one cell and give it the attribute colspan="2":

    <table>
      <thead>
        <tr>
          <th colspan="2">My Fantasy Team</th>
        </tr>
        <tr>
          <th>Player</th>
          <th>Position</th>
        </tr>
      </thead>
      <tbody>
        <tr>
          <td>Andrew Luck</td>
          <td>Quarterback</td>
        </tr>
        <tr>
          <td>Wes Welker</td>
          <td>Wide Receiver</td>
        </tr>
        <tr>
          <td>Matt Forte</td>
          <td>Running Back</td>
        </tr>
      </tbody>
    </table>

which produces

My Fantasy Team
Player Position
Andrew Luck Quarterback
Wes Welker Wide Receiver
Matt Forte Running Back

You can similarly "merge" a cell with cells below in the same column, but you use the rowspan="" attribute instead. Then make sure you have one less <td> in the rows below (however many you chose to span) so that everything still adds up right.
